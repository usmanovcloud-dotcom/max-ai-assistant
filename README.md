# MAX AI Assistant

Безопасное тестируемое ядро личного ассистента для отдельного пользовательского аккаунта MAX.

Локальный Gate 0 и текстовый AI-MVP пройдены. Проект включает приватную web-панель управления, реальный транспорт PyMax 2.3.1 и Responses API. Основной профиль — официальный OpenAI с экономичной моделью `gpt-5.6-luna`; OpenRouter сохранён как дополнительный провайдер. Для NAS поддерживается установка готового GHCR-образа через отдельный compose-файл.

## Что уже есть

- заменяемый интерфейс транспорта;
- атомарная одноразовая привязка владельца по хешированному claim-коду;
- строгая проверка sender ID и ожидаемого личного чата;
- SQLite для владельца, pairing, message ID, истории и настроек модели;
- дедупликация до внешнего запроса;
- сохранение готового ответа для повторной отправки без повторного LLM-вызова;
- последовательная очередь на чат;
- безопасное разбиение длинных ответов;
- provider interface без реальных сетевых вызовов;
- unit-тесты без MAX и LLM.
- PyMax WebClient с явно выключенной telemetry, reconnect и постоянным web user-agent;
- локальный QR-файл без вывода login URL в лог;
- отбрасывание сообщений без sender ID, из групп, каналов и от собственного аккаунта.
- OpenAI Responses API с `store=false`, ручной локальной историей и обезличенным `safety_identifier`;
- timeout, контролируемые retry, лимиты запросов/токенов и ограничение истории;
- `/new`, `/help`, `/status` без раскрытия секретов.
- локальная web-панель на `127.0.0.1` без публичного порта;
- управление API-ключами без возможности прочитать сохранённый секрет;
- web-диалоги, отправка сообщений владельцу в MAX и управление AI-процессом;
- статистика запросов, моделей, токенов, задержки и стоимости;
- безопасный журнал, аудит и backup без ключей и MAX session.
- каталог доступных версий моделей из API выбранного провайдера;
- официальный OpenAI Organization Costs API через отдельный Admin key;
- отключаемый по умолчанию вывод ответов в n8n webhook.
- безопасная диагностика ошибок провайдера по HTTP status/code/type без вывода ключей, metadata и текста переписки.

## Локальная web-панель

Запуск:

```powershell
.\.venv\Scripts\python.exe -m app.main web
```

Откройте `http://127.0.0.1:8765`. Панель по умолчанию автоматически запускает MAX AI внутри того же процесса. Отдельно запускать команду `ai` одновременно нельзя.

В панели доступны:

- статус MAX, AI, модели, очереди и последней ошибки;
- запуск, остановка и перезапуск ассистента;
- раздельное добавление OpenAI project key, опционального OpenAI Admin key и OpenRouter key;
- изменение модели, лимитов, истории, таймаутов и системной инструкции;
- отдельные web-диалоги с экспортом и очисткой;
- отправка сообщений в личный чат владельца MAX;
- QR/2FA для повторной авторизации и локальный отзыв MAX session;
- статистика, маскированные журналы, аудит и безопасная резервная копия.
- выбор доступной версии модели из живого каталога провайдера;
- процент месячного бюджета OpenAI и остаток локального дневного лимита;
- переключатель и проверка n8n webhook.

Изменяющие запросы защищены проверкой same-origin и специального заголовка. Привязка к `0.0.0.0` разрешается только в явно включённом контейнерном режиме. Обычный compose публикует порт на `127.0.0.1`, а NAS-вариант — в доверенную локальную сеть. Не пробрасывайте порт панели в интернет без отдельной аутентификации и TLS.

## Локальная проверка

Требуется Python 3.10+.

```powershell
python -m unittest discover -s tests -v
python -m app.main init-db
```

Для установки зафиксированных runtime-зависимостей:

```powershell
python -m pip install --require-hashes -r requirements.lock
```

`requirements.lock` предназначен для целевого контейнера CPython 3.12 / Linux x86_64 и фиксирует версии и SHA-256 каждого wheel.

## Одноразовая привязка (будущий Gate 0)

Claim-код создаётся только локально административной командой. В MAX владелец отправляет:

```text
/claim <случайный-длинный-код>
```

Код хранится только как SHA-256, имеет срок действия и удаляется после первого успешного claim. Повторная привязка требует локального административного сброса, который ещё не реализован.

## Локальный запуск Gate 0

```powershell
.\.venv\Scripts\python.exe -m app.main gate0
```

При первой авторизации QR сохраняется в `data/login.svg`. Не публикуйте этот файл и не пересылайте содержащуюся в нём ссылку. После успешного входа QR удаляется автоматически.

Если MAX требует 2FA, в отдельном локальном PowerShell выполните `.\.venv\Scripts\python.exe -m app.main provide-2fa`. Ввод скрыт; пароль временно сохраняется в `secrets/max-2fa.txt`, читается процессом ровно один раз и сразу удаляется. Никогда не присылайте пароль в чат.

Если владелец ещё не привязан, одноразовая команда создаётся в `data/claim-command.txt`. Отправьте её основным аккаунтом в личный чат аккаунта ассистента. После успешной привязки файл удаляется. Обычные сообщения до claim игнорируются.

Остановить процесс можно через `Ctrl+C`. Файл `data/max-session.sqlite3` содержит рабочую сессию MAX в открытом виде и не должен попадать в Git или незашифрованный backup.

## OpenAI и статистика расходов

Рабочий OpenAI project key добавляется в панели или локальной командой:

```powershell
.\.venv\Scripts\python.exe -m app.main provide-openai-key
```

Ключ используется только для Responses API и получения доступного каталога моделей. Для процента расходов обычному project key не выдаются административные права. Если нужна финансовая сводка, отдельно добавьте в панели OpenAI Admin key и укажите месячный бюджет. Панель обращается к официальному `/v1/organization/costs`; Admin key не используется для диалогов.

## n8n

Интеграция выключена по умолчанию. В разделе `n8n` укажите Production Webhook URL, при необходимости Bearer token, проверьте соединение и только затем включите переключатель. При включении в webhook передаются `event_id`, источник (`max` или `web`), провайдер, модель, prompt и response. Ошибка n8n записывается в аудит, но не блокирует ответ пользователю. URL и token хранятся в `secrets/`, не отображаются полностью и не входят в backup.

## Дополнительный OpenRouter

Используется официальный OpenRouter Responses API. Не отправляйте API-ключ в MAX, чат поддержки или переписку с разработчиком. Если ключ был раскрыт, удалите его в настройках OpenRouter и создайте новый.

Сохраните ключ локально скрытым вводом:

```powershell
.\.venv\Scripts\python.exe -m app.main provide-openrouter-key
```

Ключ попадёт только в `secrets/openrouter-api-key.txt`, который исключён из Git. Затем запустите AI-режим:

```powershell
.\.venv\Scripts\python.exe -m app.main ai
```

При переключении профиля на OpenRouter можно выбрать `openrouter/free` или конкретную доступную модель. Все значения меняются через web-панель или переменные `LLM_*`.

Официальные основы реализации: [OpenRouter Responses API](https://openrouter.ai/docs/api/reference/responses/overview), [Free Models Router](https://openrouter.ai/docs/guides/routing/routers/free-router), [лимиты OpenRouter](https://openrouter.ai/docs/api/reference/limits).

## Docker

Подготовлен рабочий compose-профиль с non-root пользователем, read-only root filesystem, отключёнными capabilities, healthcheck и постоянными каталогами `data/` и `secrets/`:

```powershell
docker compose build
docker compose up -d
docker compose ps
```

Панель будет доступна на `http://127.0.0.1:8765`. Перед запуском каталоги `data/` и `secrets/` должны быть доступны UID/GID `10001:10001`. Compose пока не разворачивается на NAS и не открывает панель в локальную сеть. Для проверки конфигурации без постоянного сервиса используется `docker compose --profile check run --rm stage1-check`.

Для Synology подготовлен отдельный [`docker-compose.nas.yml`](docker-compose.nas.yml): он не собирает проект на NAS, а скачивает готовый образ `ghcr.io/usmanovcloud-dotcom/max-ai-assistant:latest` и создаёт постоянные именованные volumes. Образ автоматически тестируется и публикуется workflow [`.github/workflows/docker-publish.yml`](.github/workflows/docker-publish.yml). Полная установка одним файлом описана в [`NAS_ONE_YAML.md`](NAS_ONE_YAML.md).

## Секреты

Не добавляйте в Git `.env`, SQLite-файлы, MAX session DB, телефон, token, device ID, 2FA-данные, API-ключи и runtime-логи. Каталоги `data/` и `secrets/` отслеживают только `.gitkeep`.
